# 文档层次结构

本文档描述了SysML v2 LSP项目的文档组织方式，采用从高级概念到具体实现的层次结构，便于用户根据需求在不同抽象级别之间导航。

## 层次结构概览

文档按照以下层次组织：

1. **架构概览** (`docs/architecture/`) - 高级系统架构和设计原则
2. **模块文档** (`docs/modules/`) - 系统主要模块和组件概览
3. **执行流程** (`docs/execution-flows/`) - 系统关键操作的详细流程
4. **函数级别** (`docs/functions/`) - 具体函数和方法的实现细节

## 导航方式

用户可以通过以下方式在层次结构中导航：

- **侧边栏**：提供完整的文档层次结构视图
- **面包屑导航**：显示当前页面在层次结构中的位置
- **相关链接**：在文档页面底部提供向上和向下的导航链接

## 文档映射

| 层次 | 文档目录 | 描述 |
|------|----------|------|
| 架构 | `docs/architecture/` | 系统总体架构、设计原则、技术栈概览 |
| 模块 | `docs/modules/` | 各主要模块（解析器、LSP工作器、Monaco编辑器集成等）概览 |
| 执行流程 | `docs/execution-flows/` | 关键用例（跳转定义、查找引用、语法高亮等）的详细流程 |
| 函数级别 | `docs/functions/` | 具体类、函数、方法的实现细节和接口说明 |

## 如何使用此结构

1. **从高级开始**：如果您是新用户，建议从[架构概览](./architecture/architecture-overview.md)开始了解系统总体设计
2. **按需深入**：根据您的兴趣点，可以逐层深入到具体的模块、执行流程或函数级别实现
3. **快速定位**：如果您要解决特定问题，可以直接跳转到相关的函数级别文档，然后通过向上导航理解其在更大 context中的作用
