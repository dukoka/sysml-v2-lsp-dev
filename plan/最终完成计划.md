- textDocument/didOpen、didChange、didClose（文档同步）
- textDocument/completion
- textDocument/hover
- textDocument/definition
- textDocument/references
- textDocument/rename（或 workspace/executeCommand 重命名）
- textDocument/documentSymbol
- textDocument/foldingRange（若 LSP 支持）或由 documentSymbol 推导
- textDocument/semanticTokens（或 document/semanticTokens/full 等）

- textDocument/signatureHelp

## 八、阶段 G、H 实施准备

以下为实施阶段 G、H 前的准备要点，便于按顺序开工。

### 阶段 G：多文件与工作区

**目标**：虚拟工作区（Map<uri, text>）+ 每文档 Index；Scope/引用解析跨 URI；跳转/引用/重命名可跨文件。

**涉及模块与顺序建议**：
1. **Worker 文档与索引**：当前已用 `TextDocuments` 管理多文档；需为每个 URI 维护「解析结果 + Scope + 符号索引」（可封装为 `DocumentIndex` 或单例 `IndexManager`），在 didOpen/didChange 时更新。
2. **Scope/引用跨 URI**：`scope.ts` / `references.ts` 扩展为可接受「当前 uri + 根 AST」与「全局 Index」（各 uri → 根 AST / Scope 根）；`resolveToDefinition` 在本地 scope 未命中时查 Index 中其他文档的导出符号（按名称或 qualifiedName）。
3. **定义/引用/重命名**：`getDefinitionAtPosition`、`findReferencesToDefinition` 返回 `Location[]` 时可带不同 uri；Monaco 侧需支持多 uri 的 Location 展示与跳转。
4. **UI 与虚拟工作区**：若需多 tab/多 URI，CodeEditor 或上层需维护「当前工作区 URI 列表」并同步到 Worker（didOpen/didClose）；单文档时可先做「单 URI + Index 结构就绪」为最小闭环。

**验收要点**：Worker 内多文档索引更新正确；跨文件跳转定义/查找引用（至少 2 个虚拟 URI）可验证；重命名跨文件生效。

### 阶段 H：LSP 为能力唯一源

**目标**：Worker 内 Connection 注册 §1.4「LSP 能力清单」全部能力；CodeEditor 仅通过 lspClient 请求并展示，Monaco 不再以本地 Provider 为主。

**涉及模块与顺序建议**：
1. **Worker 能力补全**：在 `sysmlLSPWorker.ts` 中为 definition、references、rename、documentSymbol、foldingRange、semanticTokens、signatureHelp、codeAction（及可选 selectionRange、workspace/symbol）实现 handler，并统一使用现有解析/Scope/引用/校验逻辑（与 index.ts 当前实现对齐）。
2. **lspClient 扩展**：为上述新能力在 `lspClient.ts` 增加请求方法（如 getDefinition、getReferences、getRename、getDocumentSymbols 等），与 Worker 的 LSP 方法一一对应。
3. **CodeEditor 切换**：逐步将 CodeEditor 中「来自 Monaco registerXxxProvider」的调用改为「先请求 lspClient，再把结果转成 Monaco 所需格式并展示」；最终移除或关闭对 `registerSysmlv2Language` 内各 Provider 的依赖（或仅保留词法/语言配置）。
4. **回归与开关**：可保留「LSP 未就绪时回退到本地 Provider」的开关，便于对比与回退。

**验收要点**：补全、悬停、跳转、引用、重命名、文档符号、折叠、语义高亮、格式化、诊断、签名帮助、代码操作均经 LSP 返回；Monaco 仅作 UI；测试与构建通过。

**依赖**：阶段 H 依赖阶段 G 就绪（若计划要求跨文件能力经 LSP 提供）；也可先做「单文件 + LSP 唯一源」再扩展跨文件。
